问题

发布新版本的应用程序时，可能会问你一个问题:

客户:“好的，那么自从上次发布以来你实际上做了什么？”

你:“嗯……很多！”

你可能做了很多改变/特性，但是你想要更多的信息。

另一个问题包括无法跟踪应用程序的哪些实例提供了哪些功能。你的最新作品包括所有的新功能吗？这个bug修复已经发布到测试环境了吗？嗯，现在我必须挑选所有这些提交…

解决办法

要解决这些(以及更多)问题，您可以做的事情很少:

在每个版本上更新你的应用程序——你可以很容易地从一个部署的版本跳到另一个，

  • 增加提交的可读性，
  • 创建一个changelog来跟踪自上次发布以来的更改。
  • Create a changelog to track changes since last release.

You can use the https://www.conventionalcommits.org/en/v1.0.0/ convention to make sure your commits are readable and unified.
